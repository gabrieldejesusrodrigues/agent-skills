# Product Specialist Agent — Detailed Evaluation Criteria

## Scoring Rubric (0-100)

### Problem-Solution Fit (0-25 points)
- **20-25**: Acute, frequent pain point with clear evidence of demand (forums, complaints, workarounds)
- **15-19**: Real problem but infrequent or low severity
- **10-14**: Nice-to-have; users have acceptable alternatives
- **0-9**: Solution looking for a problem

### UX Clarity & Simplicity (0-20 points)
- **16-20**: Core value deliverable in under 3 clicks/interactions; obvious onboarding
- **11-15**: Moderate learning curve; requires some explanation
- **6-10**: Complex workflow; users need training or documentation
- **0-5**: Unclear how a user would even start

### Differentiation (0-20 points)
- **16-20**: Structural moat (network effects, data advantage, unique integration)
- **11-15**: Meaningful but replicable differentiation
- **6-10**: Superficial differences (UI polish, minor feature additions)
- **0-5**: Commodity; indistinguishable from existing solutions

### Defensibility (0-15 points)
- **12-15**: Strong switching costs, data network effects, or ecosystem lock-in
- **8-11**: Moderate switching costs or brand potential
- **4-7**: Low barriers; easily cloned by incumbents
- **0-3**: No defensibility; a feature for an existing platform

### MVP Feasibility (0-20 points)
- **16-20**: Single developer can ship working MVP in 30 days with standard tools
- **11-15**: MVP achievable in 60 days; requires some integrations
- **6-10**: MVP requires 90+ days or specialized expertise
- **0-5**: Not feasible as MVP; requires extensive infrastructure or partnerships

## Engineering Complexity Rating

- **Low**: Standard CRUD app, API integrations, basic auth. One developer, 2-4 weeks.
- **Medium**: Real-time features, complex data pipelines, multiple third-party integrations. 4-8 weeks.
- **High**: ML/AI beyond API calls, complex state management, regulatory compliance requirements, multi-tenant architecture. 8-12+ weeks.

## Feature vs Company Test

A **feature** if:
- Solves a narrow problem within an existing platform's scope
- No independent pricing power
- Incumbents could add this in a sprint
- No standalone distribution channel

A **company** if:
- Addresses a workflow broad enough for independent monetization
- Creates its own data or network effects
- Requires dedicated go-to-market
- Users would pay for it standalone
